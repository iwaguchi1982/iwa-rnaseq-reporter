# Walkthrough - v0.20.2 Review Table / Filter / Search

This milestone introduced the interactive table layer for researchers to triage consensus comparisons. It provides a dedicated workspace within the Streamlit app to filter and search through comparisons before performing detailed reviews.

## Key Accomplishments

### 1. Interactive Review Table UI
Added a new section **"16. Consensus Review Table"** to the Streamlit app:
- **Bundle Intake**: Allows loading a consensus bundle manifest directly in the section.
- **Summary Dashboard**: Displays real-time metrics (Total, Filtered, Consensus, Conflict, etc.) that update as filters are applied.
- **Multi-faceted Filtering**:
    - **Decision Status**: Multiselect filter for consensus, no_consensus, etc.
    - **Decided Label**: Multiselect filter for the resulting labels.
    - **Conflict/Weak Support Modes**: Dedicated dropdowns to isolate problematic comparisons.
    - **Free Text Search**: Fast substring matching against pre-computed search indices.

### 2. Formalized Table Logic (Builder Pattern)
Separated UI from logic using a dedicated builder:
- **`ComparatorReviewTableContext`**: A pure dataclass capturing the state of a filtered table view.
- **`build_comparator_review_table_context`**: Efficient filtering logic that respects the deterministic order from v0.20.1.
- **`build_comparator_review_table_dataframe`**: Transformation logic for presentation, including label fallbacks and margin formatting.

### 3. App Integration & Compatibility
- Integrated the review workspace as an additive section in `app.py`.
- Restored `_try_load_bundle` as a compatibility wrapper to ensure legacy integration points remain functional (addressing internal test requirements).
