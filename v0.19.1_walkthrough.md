# v0.19.1: Schema Version / Provenance Spec Walkthrough

This milestone implements the **Hardening & Provenance Layer**. It adds formal schema identity and execution lineage to the consensus artifacts, enabling robust validation and auditability for downstream applications.

## Key Accomplishments

### 1. Artifact Identity (Schema Versioning)
- Added `schema_name` and `schema_version` (fixed at `0.19.1`) to both `consensus_manifest.json` and `consensus_handoff_contract.json`.
- This allows downstream tools to explicitly identify the format and required validation rules.

### 2. Execution Lineage (Provenance)
- Implemented a structured `ProvenanceSpec` that captures:
    - `producer_app` & `producer_version`: The software that created the artifact.
    - `source_consensus_run_id`: The specific execution ID.
    - `generated_at`: Accurate, timezone-aware ISO 8601 timestamp.
- This information is now embedded in all primary JSON exports.

### 3. Reporting Consistency
- Updated the human-readable `report_summary.md` to include a **Data Provenance** footer, providing immediate context for developers and clinicians reviewing the bundle manually.

### 4. Zero-Break Integration
- All changes were implemented as **additive fields** with defaults.
- Existing tools consuming the v0.18.x structure will continue to work without modification.
