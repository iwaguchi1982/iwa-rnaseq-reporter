# v0.18.2: Reference-level Ranking Walkthrough

This milestone implements the **Integrated Ranking Layer**, which calculates a unified score (0.0 to 1.0) and ranks available reference matches for each experimental comparison. It also identifies borderline cases (near-ties) that require formal consensus labeling in the next phase.

## Key Accomplishments

### 1. Weighted Integrated Scoring
- Implemented `compute_integrated_ranking_score` applying the following weights:
    - **Correlation**: 0.30
    - **Top-N Overlap**: 0.30
    - **Concordance**: 0.20
    - **Overlap**: 0.20
- This provides a single balance of evidence for each match.

### 2. Comparison-centric Ranking
- References are ranked strictly within each experimental comparison (`comparison_id`).
- Primary sort key: `integrated_score` (descending).
- Secondary sort key: `correlation_score` (descending) to resolve close matches fairly.

### 3. Conflict Detection (Near-tie Handling)
- Introduced `ComparatorTopRankConflictSpec` to record cases where the top rank is not unique.
- **Tie Tolerance**: 0.02. If the difference between top scores is less than this value, it is flagged as a `near_tie_at_top`.
- Flagging candidates with `ranking_flags` (`top_rank`, `tie_with_top`, `near_tie_with_top`).
