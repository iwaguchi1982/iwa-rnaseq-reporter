# v0.18.3: Consensus Labeling & Evidence Profiles Walkthrough

This milestone implements the **Interpretation Layer**, which synthesizes raw reference rankings into cohesive biological label candidates. It formalizes automated decision-making by evaluating the margin of support across different reference findings and identifying conflicting evidence.

## Key Accomplishments

### 1. Label Projection & Enrichment
- Enhanced `ReferenceComparisonSpec` with optional `consensus_label_key` and `consensus_label_display` fields.
- Implemented a robust resolver that falls back to raw comparison metadata if explicit consensus labels are not defined in the registry.

### 2. Evidence Aggregation (Label Grouping)
- Ranked references are now grouped by their shared biological interpretation (e.g., all references pointing to "Tumor-like" state are aggregated).
- Computed `mean_integrated_score` per label to evaluate the strength of a interpretation candidate.

### 3. Automated Consensus Decision Logic
- Implemented **Method A** for decision status:
    - **consensus**: Clear lead for a single label candidate (margin >= 0.05).
    - **no_consensus**: Competing labels are too close, or a top-rank conflict was detected in the ranking phase.
    - **insufficient_evidence**: Not enough supporting references or low scores.
- This ensures the system "abstains" from making definitive claims when evidence is weak or contradictory.
