# v0.18.1: Ranking Input Normalization / Eligibility Walkthrough

This milestone establishes the **Ranking Pre-processing Layer**, which transforms raw statistical metrics into normalized indicators (0.0 to 1.0) and filters matches based on evidence quality. This is the foundational step for the subsequent biological consensus ranking (v0.18.2).

## Key Accomplishments

### 1. Data Model for Ranking Context
- Introduced `ComparatorRankingInputContext` to hold classified and normalized results.
- Separated `ComparatorRankableMatchSpec` (evidence-eligible) from `ComparatorNonRankableMatchSpec` (evidence-weak).

### 2. Statistical Normalization
- Implemented `comparator_score_normalizer.py` with consistent scaling:
    - **Correlation**: Mapped from `[-1, 1]` to `[0, 1]`.
    - **Overlap**: Scaled by a configurable cap (default 500).
    - **Top-N Overlap**: Scaled by Top-N (default 100).
    - **Concordance**: Preserved as is (already 0-1).

### 3. Eligibility Screening (Consensus Guardrails)
- Implemented logic to exclude results with:
    - Zero gene overlap.
    - Zero Top-N overlap.
    - Missing correlation or concordance metrics.
- This ensures that only high-quality comparisons contribute to the final biological decision.
