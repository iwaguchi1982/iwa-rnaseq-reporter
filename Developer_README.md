# Developer_README.md

## 目的

この文書は `iwa-rnaseq-reporter` の開発者向けメモです。  
利用者向け README ではなく、将来の自分が設計意図と現在の実装位置を思い出すための短い整理です。

---

## 1. このアプリの役割

`iwa-rnaseq-reporter` は RNA-Seq Suite における **確認・比較設計・解析入口** のアプリです。

表向きには、Wet 利用者が

- counter の結果を読む
- サンプル構成を確認する
- QC を見る
- 発現量を確認する
- 比較したい群を整理する

ためのアプリです。

裏側では、

- comparison 解決を独立責務として扱う
- 解析実行前の中間表現を安定化する
- 結果 / report へつながる入口を作る

ことを担います。

---

## 2. Wet First の原則

このアプリも最初から **Wet 主体利用** を前提にしています。

そのため、

- README は Web UI 説明を主役にする
- comparison rule や Spec の話を前面に出しすぎない
- 利用者には「結果を見る」「比較を整理する」アプリとして見せる
- Dry 的な契約や中間表現は裏で支える

ことを重視します。

---

## 3. 現在の役割の一言整理

- counter: 結果を作る
- reporter: 結果を読んで確認し、比較の準備をする

この整理を崩さないことが重要です。

reporter は比較解析側に入るが、Wet 利用者の体感としては  
「結果確認と比較準備のアプリ」として見せる。

---

## 4. 現在の処理の大まかな流れ

内部では、おおまかに次の流れで処理します。

1. 結果を読む
2. 比較対象を決める
3. 解析を実行する
4. 出力を書き出す

この役割分離を保つことを重視します。

---

## 5. comparison 解決の考え方

### 5.1 現在の芯
comparison 解決は、解析本体から分けて扱う。  
これが現在の重要方針。

### 5.2 いまの最小安定線
現時点では主に以下を安定対象とする。

- two_group 比較
- group 解決
- sample metadata を使った群整理
- matrix と metadata の整合確認

### 5.3 将来拡張
以下は今後の拡張対象。

- sample selector
- paired design
- covariates
- comparator 連携
- signature scorer 連携

---

## 6. 入力の見せ方と内部実態

### 6.1 表向き
利用者には「counter の結果を読み込むアプリ」として見せる。

### 6.2 裏側
内部では実質的に、以下を協調利用する。

- matrix
- comparison 条件
- sample metadata

ただし README の主役にはしない。  
表では「結果」「サンプル情報」「比較したい群」という言い方で十分。

---

## 7. 出力の位置づけ

reporter は最終的に

- 解析結果
- 可視化用出力
- report 側へ渡せる構造
- 実行記録

につながる位置にある。

ただし現時点では、まず

- 読める
- 比較群を整理できる
- 次段へ進める

ことを優先する。

---

## 8. いまの設計上の芯

### 8.1 README に Spec を出しすぎない
利用者は Spec を知らなくてよい。  
内部契約は別文書で持つ。

### 8.2 comparison 解決を独立責務にする
runner に全部詰め込まず、comparison 解決は分けて扱う。

### 8.3 Wet の人に Dry 手続きを押しつけない
比較群整理の内部契約や入力整合は裏で支える。  
利用者には「画面で確認して進められる」体験を優先する。

### 8.4 日本標準の Dry を前提にする
世界標準の専業 Dry ではなく、  
「少し PC に覚えのある Wet 寄りの人」でも破綻しない説明・運用を意識する。

---

## 9. v0.3.0 での達成事項と今後の拡張メモ

### v0.3.0 での達成
- two_group 比較の安定化とエラーハンドリングの改善
- 統計的検定（Welch's t-test / BH FDR）に基づく DEG 最小実務フローの構築と結果テーブル出力の実装
- counter からの matrix とメタデータの確実な連携と読み込み

### 今後の拡張メモ
- 比較条件のさらなる拡張（sample selector, paired design, covariates）
- report export 機能の拡充と report automation との接続
- comparator 連携および signature scorer 連携
- result / payload 系の契約整理
