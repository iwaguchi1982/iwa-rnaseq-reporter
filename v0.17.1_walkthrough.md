# Walkthrough: v0.17.1 Comparator Intake / Validation Layer の導入

v0.17 マイルストンの第一歩として、Portfolio Export Bundle を正式な入力として受け取り、解析の実行可能性を検証する「Comparator Intake / Validation Layer」を実装しました。これにより、Reporter の内部状態に依存することなく、エクスポートされた成果物パッケージのみを起点とした解析オーケストレーションが可能になります。

## 実施した変更

### 1. デシリアライズ機能の拡充
- **[deg_handoff_contract.py](file:///home/manager/iwa_bio_analysis_orchestra/iwa_rnaseq_reporter/src/iwa_rnaseq_reporter/app/deg_handoff_contract.py)** / **[comparison_portfolio_handoff.py](file:///home/manager/iwa_bio_analysis_orchestra/iwa_rnaseq_reporter/src/iwa_rnaseq_reporter/app/comparison_portfolio_handoff.py)**: JSON (dict) から typed spec を復元する `from_dict` クラスメソッドを追加しました。これにより、 bundle からの読み込みが容易になります。

### 2. Comparator Intake 名前空間の確立
疎結合な解析コンシューマーとして、以下のコンポーネントを新規実装しました。
- **[comparator_intake.py](file:///home/manager/iwa_bio_analysis_orchestra/iwa_rnaseq_reporter/src/iwa_rnaseq_reporter/app/comparator_intake.py)**: 受理された比較、拒絶された比較、および検証課題（Issues）を定義するデータモデル。
- **[comparator_bundle_reader.py](file:///home/manager/iwa_bio_analysis_orchestra/iwa_rnaseq_reporter/src/iwa_rnaseq_reporter/app/comparator_bundle_reader.py)**: ZIP アーカイブからマニフェストや契約ファイルを安全に抽出する読み込み層。
- **[comparator_intake_builder.py](file:///home/manager/iwa_bio_analysis_orchestra/iwa_rnaseq_reporter/src/iwa_rnaseq_reporter/app/comparator_intake_builder.py)**: バンドル全体の整合性を検証し、比較可能性に基づいて `Accepted` / `Rejected` を仕分けるバリデーション・ロジック。

### 3. 解析準備完了 (Readiness) 判定の導入
解析対象の比較群において、`matrix_kind` や `feature_id_system` が一貫しており、かつ不明なメタデータ（unknown 等）が含まれていない場合にのみ `is_ready_for_reference_matching: True` と判定するガードロジックを実装しました。
