# Reporter Handoff Profile (v0.12)

This document defines the formal data contract between `iwa_rnaseq_counter` and `iwa_rnaseq_reporter`. The objective is to ensure that the Reporter can consume Analysis Bundles using established helper libraries, without ever having to parse raw JSON files or manually handle complex nested structures.

## 1. Design Principles

- **Black Box Consumption**: The Reporter treats the Analysis Bundle as a "Black Box". It only interacts with the bundle through the `iwa_rnaseq_counter` public API.
- **Surface Optimization**: The Handoff Profile exposes only the fields necessary for reporting, visualization, and comparison resolution.
- **Non-Validating Consumption**: The Reporter assumes the bundle has already been validated by the Counter CLI (`validate-analysis-bundle`). It uses "Safety Accessors" to retrieve values with reliable defaults.

## 2. Mandatory Fields (Required)

These fields are essential for the Reporter to load data and identify the run.

| Field | Description | Counter Helper / Accessor |
| :--- | :--- | :--- |
| `analysis_bundle_manifest_path` | Absolute path to the bundle entry point. | `get_bundle_manifest_path()` |
| `run_id` | Unique identifier for the analysis run. | `get_bundle_run_id()` |
| `matrix_id` | Identifier for the processed expression matrix. | `get_bundle_matrix_id()` |
| `matrix_shape` | Dictionary with `feature_count` and `sample_count`. | `get_bundle_matrix_shape()` |
| `sample_axis` | The axis representing samples (typically `specimen`). | `get_bundle_sample_axis()` |
| `feature_id_system` | The ID system used for features (e.g., `ensembl_gene_id`). | `get_bundle_feature_id_system()` |
| `column_order_specimen_ids` | Ordered list of sample IDs representing the matrix columns. | `get_bundle_column_order_specimen_ids()` |

## 3. Optional & Descriptive Fields

These fields are used for UI display, warnings, and traceability.

| Field | Description | Counter Helper / Accessor |
| :--- | :--- | :--- |
| `producer` / `version` | Information about the pipeline that created the bundle. | `summarize_analysis_bundle_for_consumer()` |
| `source_quantifier_summary` | Details about the quantifier tool (e.g., Salmon, Kallisto). | `get_bundle_source_quantifier_summary()` |
| `feature_annotation_status` | Status of the feature annotation merge (if any). | `get_bundle_feature_annotation_status()` |
| `sample_metadata_alignment_status` | Status of the sample metadata join. | `get_bundle_sample_metadata_alignment_status()` |
| `warning_summary` | High-level summary of warnings recorded during merge. | `get_bundle_warning_summary()` |

## 4. Consumer Workflow

The Reporter implementation of the bundle loader should follow this pattern:

```python
from iwa_rnaseq_counter.io.read_analysis_bundle import (
    read_analysis_bundle,
    summarize_analysis_bundle_for_consumer
)

def load_bundle_for_reporter(manifest_path: str):
    # 1. Load the bundle via counter library
    bundle = read_analysis_bundle(manifest_path)
    
    # 2. Extract the Handoff Profile summary
    # This dictionary corresponds directly to the fields above.
    handoff_summary = summarize_analysis_bundle_for_consumer(bundle)
    
    # 3. Initialize the Reporter View Model
    return ReporterAnalysisBundle(**handoff_summary)
```

By following this profile, the Reporter side remains decoupled from the internal JSON schema of the Analysis Bundle.
