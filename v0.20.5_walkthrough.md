# Walkthrough - v0.20.5 Review Export / Handoff

This milestone completes the v0.20 series by enabling researchers to export their review results as a formal, packaged bundle. This "Review Bundle" serves as the authoritative record of human triage and notes, ready for downstream reporting or data integration.

## Key Accomplishments

### 1. Formal Review Bundle (ZIP)
Introduced a packaging system that aggregates all review-related data:
- **`review_manifest.json`**: Tracking the review ID, source consensus ID, and aggregate progress.
- **`review_rows.json` / `review_rows.csv`**: Full export of all comparisons (including unreviewed ones) paired with their manual triage states and notes.
- **`review_summary.json` / `review_summary.md`**: Machine and human-readable progress reports.
- **`review_handoff_contract.json`**: A compact, downstream-ready contract listing all final reviewer judgments.

### 2. Built-in Accountability (Provenance)
- The review bundle captures a **Source Refs Snapshot**, documenting exactly which consensus bundle (and specific internal JSON artifacts) was reviewed.
- Every bundle is given a unique `review-run-id` based on timestamps to prevent identification collisions.

### 3. Review Workspace Integration
Added a **Review Export & Handoff** section at the bottom of the workspace:
- **Instant Download**: High-speed, in-memory ZIP generation for the active session.
- **Integrated Previews**: Multi-tab JSON previews for the Manifest, Summary, and Handoff Contract.
- **Markdown Report Preview**: At-a-glance view of the final summary report before downloading.

### 4. Robust Import & Validation
- Implemented a validator to ensure imported review bundles are structurally sound and consistent with their source consensus runs.
- Verified the integrity of the roundtrip: `Export -> ZIP -> Import == Original State`.
